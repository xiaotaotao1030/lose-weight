## AI 功能说明

当前 AI 饮食识别是前端原型模拟结果，不会上传图片。

真实上线时需要增加后端接口，由后端调用视觉 AI 服务。API Key 不能写在前端代码里。

文字饮食分析只使用联网 GPT 模式，需要部署到 Vercel，并在项目环境变量中配置：

```text
OPENAI_API_KEY=YOUR_API_KEY_HERE
```

可选配置：

```text
OPENAI_MODEL=gpt-4o-mini
```

后端接口为 `POST /api/analyze-food`，返回标准结构：

```json
{
  "foods": [
    {
      "name": "",
      "quantity": "",
      "calories": 0,
      "protein": 0,
      "carbs": 0,
      "fat": 0
    }
  ],
  "total": {
    "calories": 0,
    "protein": 0,
    "carbs": 0,
    "fat": 0
  }
}
```

## 身体照片打卡说明

当前身体照片只保存在当前浏览器本地。清理浏览器缓存或换设备后，记录可能丢失。

## 每日记录说明

每日体重、饮食和运动记录保存在当前浏览器本地。切换设备或清理浏览器数据后，记录可能丢失。

## 添加到手机主屏幕

iPhone Safari 打开页面后，可以通过“分享”菜单选择“添加到主屏幕”。

当前已关闭离线缓存和 service worker 注册，页面按普通网页方式加载，避免 iPhone 主屏幕版本读取旧缓存。
